THEY POINT TO EACH OTHER!
ADDRESS-BY-ADDRESS EXPLANATION

ADDRESS 1: current

Address: 0xffff8f4dd168d400
What: struct task_struct (the process)
Created: When process was fork()'d
Contains: PID, files, memory maps, etc.
This is the kernel's representation of your socket_test process. Every process has one. The kernel macro 'current' gives you this address.

ADDRESS 2: current->files

Address: 0xffff8f4da9aba940
What: struct files_struct
Created: When process was fork()'d
Contains: File descriptor table (fdt)
This points to the file descriptor table structure. It has an array fdt->fd[] where each slot holds a struct file*.

ADDRESS 3: fdt->fd[3] (struct file)

Address: 0xffff8f4e6b37b0c0
What: struct file
Created by: sock_alloc_file() during socket()
How: Kernel called __alloc_file() from slab cache
This is the VFS layer wrapper. Every file descriptor points to one. For sockets, the key field is private_data which points to the socket.
LINK: file->private_data = 0xffff8f4e33230340
This links the file to the socket structure.

ADDRESS 4: struct socket

Address: 0xffff8f4e33230340
What: struct socket
Created by: sock_alloc() during socket()
How: Kernel allocated from socket_cache slab
This is the protocol-independent socket abstraction. It sits between VFS (struct file) and transport (struct sock).
LINK 1: socket->file = 0xffff8f4e6b37b0c0
Points BACK to struct file (bidirectional)

LINK 2: socket->sk = 0xffff8f4e2e47e880
Points DOWN to struct sock (transport layer)

ADDRESS 5: struct sock

Address: 0xffff8f4e2e47e880
What: struct sock (actually struct tcp_sock)
Created by: sk_alloc() during socket()
How: Kernel allocated from tcp_sock slab cache
This is the TCP state machine. Contains: - sk_family = 2 (AF_INET = IPv4) - sk_protocol = 6 (IPPROTO_TCP) - sk_state = 7 (TCP_CLOSE = not connected)
LINK: sock->sk_socket = 0xffff8f4e33230340
Points BACK to struct socket (bidirectional)

ADDRESS 6: socket_file_ops

Address: 0xffffffff9416e9a0
What: struct file_operations
Created: Compiled into kernel (static)
Note: 0xffffffff = kernel text segment
This is a function pointer table. When you call read(fd), the kernel looks up file->f_op->read and calls it. For sockets, this is socket_file_ops.

ADDRESS 7: inet_stream_ops

Address: 0xffffffff94183c00
What: struct proto_ops
Created: Compiled into kernel (static)
Protocol-specific operations for TCP over IPv4. Contains bind(), listen(), accept(), connect(), etc.

ADDRESS 8: tcp_prot

Address: 0xffffffff953e2400
What: struct proto (TCP protocol)
Created: Compiled into kernel (static)
The TCP implementation. Contains connect(), sendmsg(), recvmsg(). This is where the actual TCP state machine lives.
